# Events Implementation Plan (oli-theme — Cycle 1, Plan 6/10)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Add multilingual events: CPT `oli_event` with start/end dates, location, price, registration URL; immutable `EventEntity`; `EventModel` for upcoming/past queries; `EventController` (single) + `EventArchiveController` (archive); admin metabox; theme-bridge files; templates; CSS module. After this plan, the theme exposes a real event system.

**Architecture:** Mirror of Plan 3 (Posts) and Plan 5 (Slides) patterns — DTO + Model + Controller + Module + theme-bridge. Two controllers (one per render context). A Metabox class encapsulates the admin form for the event-specific meta fields. Event slug per language is supported (`/fr/evenements/`, `/en/events/`) via the existing rewrite system.

**Tech Stack:** Same as Plans 1-5.

**Reference spec:** `docs/superpowers/specs/2026-05-05-oli-theme-design.md` — sections 2.3 (CPT `oli_event`).

**Out of scope:** booking, payment, recurring events, calendar UI front (priorité 3 spec).

### T4 — `EventModel` + `EventModelInterface`

Methods:
- `findUpcoming(Language, int $limit = 10): array` (start_date >= now, ordered ASC)
- `findPast(Language, int $limit = 10): array` (end_date < now, ordered DESC)
- `findById(int): ?EventEntity`
- `findBySlug(string, Language): ?EventEntity`

Implementation: `get_posts` avec `post_type=oli_event`, `meta_query` pour `_oli_event_start_date` ou `_oli_event_end_date`. Hydrate via meta keys :
- `_oli_event_start_date` (Y-m-d H:i:s)
- `_oli_event_end_date` (idem)
- `_oli_event_location`
- `_oli_event_address`
- `_oli_event_flyer_url`
- `_oli_event_registration_url`
- `_oli_event_price`

`isPast` = `endDate < now` (ou si pas d'endDate, `startDate < now`).
`isOngoing` = `startDate <= now <= endDate` (si endDate présente).

Constructor: `(LanguageRegistryInterface $registry)`.

Tests : 5 (empty upcoming, builds upcoming, empty past, findById null/found, findBySlug). Brain Monkey stubs.

Commit: `feat(events): ajoute EventModel (upcoming/past/byId/bySlug)`.

### T5 — `EventController` (single)

Constructor: `(EventModelInterface $events, LanguageResolverInterface $resolver, LanguageSwitcherControllerInterface $switcher, MenuControllerInterface $menus, RendererInterface $renderer)`.

`renderSingle(): string` :

- if null → `pages/404` avec base view-model
- else → `pages/single-event` avec view-model contenant `event` (EventEntity), `lang`, `languageSwitcher`, `primaryMenu`, `footerMenu`, `bodyClasses`.

Tests : 2 (rendered with entity / 404 fallback).

Commit: `feat(events): ajoute EventController (rendu single)`.
