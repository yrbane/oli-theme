# Contact Form Implementation Plan (oli-theme — Cycle 1, Plan 8/10)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Build a custom OOP/TDD contact form with strict security (CSRF + honeypot + time-trap + rate limit + sanitization), multilingual labels, optional auto-reply and optional logging via CPT `oli_contact_log`. After this plan, the theme exposes a real, secure contact form usable on any page via shortcode or template inclusion.

**Architecture:** Mirror of previous modules.

- **`ContactFormModel`** — validation + sanitization (DTO `ContactSubmission` immuable + résultat de validation `ContactValidationResult`).
- **`ContactRateLimiter`** — transient-based throttling (3 envois max par 15 min par IP).
- **`ContactMailer`** — envoi via `wp_mail`, support auto-reply.
- **`ContactLogModel`** — optionnel, CPT `oli_contact_log` (titre = sujet, content = message, meta = nom/email/IP/timestamp).
- **`ContactFormController`** — orchestrateur : nonce + honeypot + time-trap + rate-limit + validation + sanitize + send + log + redirect.
- **`ContactShortcode`** — `[oli_contact_form]` rend le partial.
- **`ContactModule`** — services + hooks `admin_post_oli_contact` (POST classique) + `admin_post_nopriv_oli_contact`.
- Template `partials/contact-form.html.tpl` + CSS `assets/css/contact.css` + JS `assets/js/contact-form.js` (progressive enhancement).

**Tech Stack:** Same as Plans 1-7.

**Reference spec:** `docs/superpowers/specs/2026-05-05-oli-theme-design.md` § 4.2 (Module Contact).

### T4 — `ContactFormModel` + Interface

`validate(ContactSubmission $sub): ContactValidationResult` :
- `name` : 2-100 chars, sinon `'name_invalid'`
- `email` : `is_email()` (WP), sinon `'email_invalid'`
- `subject` : si non null, 1-150 chars, sinon `'subject_too_long'`
- `message` : 10-5000 chars, sinon `'message_invalid'`
- `honeypot` : doit être vide, sinon `'spam_detected'`
- `timestamp` vs now : doit être ≥ 3 secondes après le rendu du form, sinon `'too_fast'` (anti-bot time-trap)

`sanitize(ContactSubmission $sub): ContactSubmission` retourne une nouvelle instance avec :
- `name` via `sanitize_text_field`
- `email` via `sanitize_email`
- `subject` via `sanitize_text_field`
- `message` via `sanitize_textarea_field`
- les autres inchangés

Tests : 6 (tous les cas + sanitize).

Commit: `feat(contact): ajoute ContactFormModel (validation + sanitization)`.

### T5 — `ContactRateLimiter` + Interface

Constructor: aucune dépendance (utilise les transients WP).

`attempt(string $ip): bool` :
- Lit `get_transient('oli_contact_rate_' . md5($ip))` → un int (compteur).
- Si ≥ 3, retourne `false`.
- Sinon, incrémente et `set_transient(..., $count + 1, 15 * MINUTE_IN_SECONDS)`. Retourne `true`.

Tests : 3 (`testFirstAttemptAllowed`, `testFourthAttemptBlocked`, `testCounterIsolatedPerIp`).

Commit: `feat(contact): ajoute ContactRateLimiter (3 envois / 15 min par IP)`.

### T6 — `ContactMailer` + Interface

Constructor: aucune.

`send(ContactSubmission $sub, string $to): bool` :
- Build `$subject = '[Contact] ' . ($sub->subject ?? 'Nouveau message')`.
- Build body avec name, email, subject, message + IP + timestamp.
- Headers : `From: $siteName <$siteEmail>`, `Reply-To: $sub->name <$sub->email>`.
- Retourne `wp_mail($to, $subject, $body, $headers)`.

`sendAutoReply(ContactSubmission $sub, string $bodyTemplate): bool` :
- Subject : `Confirmation de votre message`.
- Body : `$bodyTemplate` (texte localisé fourni par le controller).
- Retourne `wp_mail($sub->email, $subject, $body, $headers)`.

Tests : 3.

Commit: `feat(contact): ajoute ContactMailer (wp_mail + reply-to + auto-reply)`.

### T8 — `ContactFormController` + Interface

Constructor: `(ContactFormModelInterface $model, ContactRateLimiterInterface $rateLimiter, ContactMailerInterface $mailer, ContactLogModel $log, LanguageResolverInterface $resolver)`.

`handle(array $postData): void` :
1. Vérifier nonce (`wp_verify_nonce($postData['_oli_nonce'], 'oli_contact')`). Si fail → `wp_die(403)`.
2. Construire `ContactSubmission` depuis `$postData` + IP serveur.
3. Si `$rateLimiter->attempt($sub->ip) === false` → redirect avec error param « rate_limit ».
4. `$validation = $model->validate($sub)`. Si `! $validation->valid` → redirect avec query params d'erreur.
5. `$sanitized = $model->sanitize($sub)`.
6. `$mailer->send($sanitized, $to)` où `$to` est l'email de destination (option `oli_contact_email`).
7. Optionnel : `$mailer->sendAutoReply(...)` si `oli_contact_autoreply` activé.
8. Optionnel : `$log->log($sanitized)` si `oli_contact_logging` activé.
9. `wp_safe_redirect($postData['_oli_redirect'] ?? home_url('/?contact=ok'))`. `exit;`.

Pour la testabilité : `wp_die`, `wp_safe_redirect`, `exit` sont mockés via Brain Monkey.

Tests : 6 (`testRejectsInvalidNonce`, `testRedirectsOnRateLimit`, `testRedirectsOnValidationError`, `testSendsEmailOnSuccess`, `testCallsAutoReplyWhenEnabled`, `testLogsWhenEnabled`).

Commit: `feat(contact): ajoute ContactFormController (orchestration sécurisée)`.
